# /fix-codesign-error

检查 macOS 签名或授权失败，并解释最小修复路径。

## 参数

- `app`: `.app` 捆绑包或二进制文件的路径（可选）
- `identity`: 签名身份提示（可选）
- `mode`: `inspect` 或 `repair-plan`（可选，默认：`inspect`）

## 工作流程

1. 检查应用捆绑包、可执行文件、签名信息和授权。
2. 确定问题是身份、配置配置文件、硬化运行时、沙盒还是信任策略。
3. 用plain语言总结确切的失败类别。
4. 提供最小修复序列或验证命令。

## 限制

- 从二进制文件或源文件读取授权，永不 inventing 授权。
- 区分本地开发签名问题与分发或公证失败。
- 相信可验证命令，如 `codesign -d`、`spctl` 和 `plutil`，而不是猜测。
