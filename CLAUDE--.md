# CLAUDE.md

此文件为 Claude Code (claude.ai/code) 在处理此代码库时提供指导。

## 核心架构概览
本项目是一个高度模块化的、可代理/可转接的 CLI/TUI 框架。其核心在于提供一个统一的交互层，该层可以灵活地将内部逻辑（基于 Anthropic Messages 结构）适配到不同的外部 API 接口，包括但不限于 OpenAI Chat Completions。

代码库的架构围绕以下几个关键模块构建：
1. **API 适配层 (`src/services/api/*`)**: 管理与外部大模型服务的连接。这是实现模型切换、API 格式兼容（如 OpenAI 兼容层）和认证的核心区域。
2. **会话管理层 (`src/commands/*`)**: 负责处理用户与模型的交互生命周期。它抽象了会话的创建、清理、回滚、压缩等高级操作，这些操作极大地增强了用户体验和上下文管理能力。
3. **功能扩展层 (Plugins/Skills/Agents)**: 采用了插件化、技能化和代理化的设计模式。高层逻辑的实现是组合式的，即通过调用预定义的“技能”或“代理”来实现复杂的业务流程，而不是在单一文件中硬编码。

理解项目流向的关键在于跟踪**用户输入 -> 状态管理 -> 命令解析 -> 选定技能/代理执行 -> API 适配 -> 模型返回 -> 状态更新**的完整流程。

## 常用操作流程
在开发或使用过程中，关注以下几个流程性命令：
- **会话维护**: 使用 `/clear` 重置上下文；使用 `/compact` 压缩冗余上下文以优化 Token 消耗；使用 `/context` 监控当前上下文的用量。
- **功能调用**: 使用 `/plugins`、`/skills` 和 `/agents` 来管理和调用核心功能模块，这些模块代表了代码库最复杂的业务逻辑。
- **初始化/诊断**: `/init` 重新审视项目结构，确保所有模块均处于最新状态。

## 开发与运行环境
该项目依赖 Bun 作为运行时环境。
1. **依赖安装**: 在项目根目录下执行 `bun install` 来同步所有依赖。
2. **开发模式**: 运行 `bun run dev` 来启动可交互的 CLI/TUI 界面。
3. **全局命令注册**: 对于需要作为全局命令调用的场景，推荐使用 `bun link` 配合 `bun install` 来确保命令入口与当前源码版本同步。

## 维护与同步
由于本项目是基于上游 Claude Code 进行深度定制和魔改的 Fork 分支，请特别注意其维护和更新流程。推荐通过 `git pull` 拉取最新代码后，务必执行 `bun install` 和 `bun link` 来同步依赖和全局命令入口。
